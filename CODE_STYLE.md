# Code Style

-   Component files: PascalCase for component names, folder names lowercase for features.
-   Hooks: prefix with use (useFetchRecipes)
-   Files: one component per file, default export for components.
-   Prefer React.memo for pure presentational components.
-   Use PropTypes or TypeScript for prop contracts.

ESLint + Prettier: configurar rules para evitar console.log en producción y forzar semicolons o no (según preferencia).
